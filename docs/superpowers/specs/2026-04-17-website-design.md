# Talks Portfolio Website — Design Spec

**Date:** 2026-04-17
**Status:** Approved for planning
**Location in repo:** `./website/`

## 1. Goals & non-goals

### Goals

1. **Amplify authority.** The site clearly positions Juraci as a speaker and engineer — OpenTelemetry Governance Committee member, co-founder of OllyGarden, Jaeger emeritus maintainer — with a visible body of work.
2. **Be instantly shareable.** Every talk has a stable URL (`/talks/<slug>/`) with rich OG/Twitter cards, schema.org JSON-LD, and semantic HTML so pasted links preview well and LLMs cite content accurately.
3. **Be discoverable.** First-class SEO (sitemap, canonical URLs, structured data) and AI SEO (`llms.txt`, LLM-readable abstracts in the HTML body, not iframes).
4. **Showcase Credly badges** from https://www.credly.com/users/juraci-paixao-krohling, with manual featuring of high-value badges.
5. **Stay low-maintenance.** Adding a new talk = drop a PDF + a markdown file with frontmatter. Rebuild. No CMS.

### Non-goals (v1)

- Full i18n of non-bio content (only the bio has a language toggle; talk pages stay English)
- Blog / CMS
- Dynamic backend, contact form, or auth
- Dark mode
- Full-text search
- Newsletter / email capture

## 2. Content model

### Directory structure (unchanged)

Talks continue to live in the repo root under year-based directories:

```
YYYY/
  YYYY-MM-DD-event-slug/
    README.md         # frontmatter + abstract + co-speakers section
    slides.pdf
```

The website (`./website/`) reads from this structure via a custom Astro content loader.

### Frontmatter schema (validated by Zod)

Each talk's `README.md` gains YAML frontmatter at the top:

```yaml
---
title: "Taming Wasteful Telemetry"
event: "KubeCon + CloudNativeCon Europe 2026"
event_slug: "kubecon-eu-2026"        # stable identifier for grouping
date: 2026-03-26                      # ISO date
location:
  city: "Amsterdam"
  country: "Netherlands"
  flag: "🇳🇱"
event_link: "https://sched.co/2EF6i"  # optional
recording: "https://youtu.be/..."     # optional; YouTube URL (short or long form). Other providers deferred to v2.
slides: "slides.pdf"                  # path relative to this dir; optional
tags: [opentelemetry, collector, cost]
featured: true                        # optional, default false
co_speakers:                          # optional
  - name: "Pablo Baeyens"
    company: "Datadog"
---
```

The markdown body below the frontmatter is the **abstract** and any rendered sections (no duplicate metadata table — frontmatter is the single source of truth).

### Migration of existing READMEs

A one-shot script `scripts/migrate-talks.ts` parses the existing table + body of each talk README and writes frontmatter above it. Must be **idempotent** (skip any file that already has frontmatter). A sample output is reviewed before bulk-running against the ~37 existing talks.

The existing table in each README is removed (frontmatter replaces it); the human-readable rendering of that metadata is regenerated at display time by the website.

### Other content collections

- **Projects** (`website/src/content/projects/*.md`): OllyGarden, OpenTelemetry, Jaeger — each with name, role, description, link, period.
- **Topics** (`website/src/content/topics/*.md`): tag slug, display name, 1-sentence description. Drives `/topics/<tag>/` pages.
- **Media kit** (`website/src/content/media-kit/*.md`): three bios (en/de/pt) + one-liner + headshot references. Initial content is copied from the existing bios in the repo's root `README.md`; the website becomes the canonical home going forward.

### Credly data

- `website/data/credly/badges.json` — generated by the build from Credly's public profile API.
- `website/data/credly/featured.json` — hand-edited: `{ "featured_order": ["badge-id-1", "badge-id-2"] }`. Badges in this list float to the top with a star treatment; everything else sorted by `issued_at` descending.

## 3. Page architecture

All pages are statically generated.

| Route | Source | Purpose |
|---|---|---|
| `/` | `src/pages/index.astro` | Split-hero homepage |
| `/talks/` | `src/pages/talks/index.astro` | Magazine archive: featured + list |
| `/talks/<slug>/` | `src/pages/talks/[...slug].astro` | Editorial talk detail |
| `/topics/` | `src/pages/topics/index.astro` | Topic cloud + per-topic counts |
| `/topics/<tag>/` | `src/pages/topics/[tag].astro` | Talks matching a tag |
| `/projects/` | `src/pages/projects.astro` | Affiliations & projects |
| `/badges/` | `src/pages/badges.astro` | All Credly badges, featured first |
| `/press/` | `src/pages/press.astro` | Media kit: 3-language bios, headshots, one-liner, copy buttons |
| `/rss.xml` | `src/pages/rss.xml.ts` | Talk feed |
| `/sitemap.xml` | `@astrojs/sitemap` | SEO |
| `/llms.txt` | `src/pages/llms.txt.ts` | LLM-friendly summary + deep links |
| `/og/<slug>.png` | `public/og/` (generated at build) | Per-talk social preview image |

**Talk slug:** `YYYY-MM-DD-event-slug` — matches the existing directory name. Stable forever; no rewrites expected.

### Layouts

- `BaseLayout.astro` — `<html>`, head (meta, OG/Twitter, fonts, canonical, favicon + apple-touch-icon), top nav (cream bg, bordeaux wordmark, links: Talks · Topics · Projects · Badges · Press), footer (linktr.ee + social + copyright). Injects `Person` JSON-LD.
- `TalkLayout.astro` — extends Base, injects `PresentationDigitalDocument` nested in `Event` JSON-LD.

### Components (focused, single-purpose)

- `HeroSplit.astro` — homepage bordeaux identity panel + featured talks list
- `TalkCard.astro` — used in archive featured row + homepage rail
- `TalkList.astro` — dense row list for archive
- `FilterChips.astro` — **client island** (interactive filter on archive)
- `BioLanguageToggle.astro` — **client island** on `/press/`
- `CredlyBadge.astro` — single badge, supports featured variant
- `TagPill.astro` — shared tag display
- `YouTubeEmbed.astro` — lite-youtube facade to keep JS light

## 4. Visual system

### Layout decisions (from brainstorm)

- **Visual direction:** Warm Narrative — bordeaux + cream + warm neutrals; serif display; approachable.
- **Homepage:** Split hero — bordeaux identity panel (name, tagline, topic chips) on the left, featured talks list on the right; below-the-fold: about, projects, badges preview.
- **Archive:** Magazine — 1 hero-sized featured talk + 2 smaller featured cards at top, dense chronological list below, filter chips for event/tags.
- **Talk page:** Editorial long-read — quiet hero (event eyebrow, title, meta row), main column with YouTube embed + abstract in serif, metadata sidebar with slides download + tags + co-speakers.

```css
  --font-body:    'Source Serif 4 Variable', Georgia, serif;
  --font-ui:      'Inter Variable', system-ui, sans-serif;

  /* Scale (8-pt grid) */

  --space-4: 1rem;   --space-6: 1.5rem; --space-8: 2rem;
  --space-12: 3rem;  --space-16: 4rem;

  /* Radii */

  --radius-pill: 100px;

```

### Typography

- **Display:** Fraunces (variable, `@fontsource-variable/fraunces`)
- **Long-form body:** Source Serif 4 (variable, `@fontsource-variable/source-serif-4`)
- **UI labels, meta, buttons:** Inter (variable, `@fontsource-variable/inter`)
- Self-hosted via Fontsource; no Google Fonts CDN.
- `font-display: swap` on all three. Preload Fraunces (used above the fold).

### CSS approach

- **No CSS framework.** Vanilla CSS + tokens + Astro's built-in component-scoped `<style>` blocks.
- Global reset + tokens + typography rules in `src/styles/global.css`, imported once from `BaseLayout.astro`.
- Per-component styles live inside each `.astro` file's `<style>` block.

### Responsive

- Mobile-first. Breakpoints: `≥640px` (sm), `≥960px` (md), `≥1200px` (lg).
- Split hero stacks on mobile.
- Archive featured row collapses to 1 col; list rows reflow to cards.
- Talk page sidebar drops below the main column.

### Accessibility

- WCAG 2.1 AA. Color contrasts verified in the implementation phase (bordeaux-700 on cream-100 is the critical pair).
- Visible focus rings (bordeaux-700, 2px, offset 2px).
- `prefers-reduced-motion: reduce` disables the minimal page transitions.
- All interactive elements are real buttons/links; filter chips are `<button>` with `aria-pressed`.
- Skip-to-content link in `BaseLayout`.

## 5. Build pipeline & SEO

### Build order (`bun run build`)

1. **`scripts/fetch-credly.ts`** — fetches the full paginated badge list from Credly's public profile API. Writes normalized JSON to `data/credly/badges.json`. On failure: log a warning and keep the previous snapshot; **never fail the build** on Credly unavailability.
2. **Astro content collection loaders** — `content.config.ts` loads:
   - Talks via Astro's built-in `glob()` loader, pattern `../YYYY/**/README.md` (resolved relative to the `website/` dir). Each talk's `slug` is derived from its directory name.
   - Projects, topics, media-kit via `glob()` against `src/content/`.
   - All collections have Zod schemas; invalid frontmatter **fails the build**.
3. **`astro build`** — static HTML generation. Slide PDFs are copied as static assets alongside their talk page.
4. **Sitemap** — `@astrojs/sitemap` integration generates `/sitemap.xml`.
5. **OG image generation** — per-talk static PNG via `satori` + `@resvg/resvg-js`. Title, event, date on a bordeaux card. Cached to `public/og/<slug>.png` (regenerated only when the talk frontmatter changes — keyed by a content hash).
6. **`llms.txt`** — custom endpoint emits a plain-text directory: bio, projects, topic → talk matrix, canonical URLs.

### SEO

- `<link rel="canonical">` on every page.
- Per-page OG + Twitter card with the generated OG image.
- Base layout emits `Person` JSON-LD (name, role, affiliations, sameAs → linktr.ee, github, linkedin).
- Talk pages emit `PresentationDigitalDocument` nested inside an `Event` node (with location, date, url).
- Archive page emits `ItemList`.
- `robots.txt` — allow all, point to sitemap.
- `hreflang` tags on `/press/` for the three bio languages.

### AI SEO

- `llms.txt` at site root, following the [llmstxt.org](https://llmstxt.org) convention.
- All abstract text is in the HTML body (not behind JS-only iframes). YouTube uses a lite-facade that falls back to a normal link.
- `<meta name="author">` + `rel="author"` link to the homepage.
- Structured data complements — does not replace — semantic HTML.

### Content pipeline

- `bun run sync:credly` can be invoked locally or via a weekly scheduled GitHub Action.
- The workflow commits any Credly JSON changes as a PR against `main` so Juraci can review new badges before they appear.

## 6. Ops, deployment & testing

### Repo layout

```
./
├── YYYY/…/README.md         # talks (source of truth for content)
├── headshots/
├── website/
│   ├── astro.config.mjs
│   ├── package.json
│   ├── bun.lockb
│   ├── tsconfig.json
│   ├── src/
│   │   ├── pages/
│   │   ├── layouts/
│   │   ├── components/
│   │   ├── content/          # projects, topics, media-kit
│   │   ├── content.config.ts
│   │   ├── lib/
│   │   └── styles/           # tokens.css, global.css
│   ├── scripts/
│   │   ├── fetch-credly.ts
│   │   └── migrate-talks.ts
│   ├── public/               # static assets (including generated og/)
│   └── data/credly/          # badges.json (generated), featured.json (hand-edited)
└── .github/workflows/
    ├── deploy.yml            # GitHub Pages deploy on push to main
    └── sync-credly.yml       # weekly Credly sync → PR
```

### Runtime & tooling

- **Runtime:** Bun (package manager + script runner)
- **Framework:** Astro (uses Vite internally)
- **Language:** TypeScript (strict)
- **Lint/format:** biome (single dependency covers both)
- **Package manager:** Bun (`bun.lockb` committed)

### Deployment (GitHub Pages)

- `.github/workflows/deploy.yml`:
  - Triggers: push to `main` touching `website/**` or `YYYY/**`
  - Steps: `bun install` → `bun run build` (in `website/`) → `actions/upload-pages-artifact` → `actions/deploy-pages`
- Custom domain optional. If `kroehling.de` is used, add a `CNAME` file to `website/public/` and configure DNS.

### Testing

Proportional to scope — no ceremony:

- **Zod frontmatter validation** — first line of defense. A missing `event` or malformed `date` fails the build.
- **Playwright smoke tests** — one per route type (home, talk detail, archive, badges, press). Checks page loads, key h1, critical meta tags present. Runs on PR via CI.
- **Lighthouse CI** — runs on PR against 4 canonical routes. Budget: perf ≥95, a11y ≥95, SEO 100.
- **Lychee link checker** — catches dead external links as talks age. Runs weekly.

No unit tests for visual components; no visual regression suite.

### Observability (post-launch, optional)

Out of scope for v1, but trivial to add:
- Plausible or Umami for analytics (self-hosted or lite-cloud, privacy-respecting).
- Post-deploy Lighthouse run archived for trending.

## 7. Out of scope for v1 (deferred)

- Contact form, newsletter signup
- Comment system on talk pages
